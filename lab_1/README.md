# Лабораторная работа №1  
## Верификация модели автомата выдачи корма для белок в nuXmv

### Тема работы
В данной лабораторной работе рассматривается система **"Автомат выдачи корма для белок"** из предложенного списка вариантов.

### Цель работы
Построить модель параллельной системы на языке **nuXmv**, сформулировать свойства на языке **LTL** и провести их верификацию.  
Согласно требованиям лабораторной работы, система должна содержать:
- процесс управляющего алгоритма;
- процессы датчиков и/или индикаторов;
- процессы актуаторов;
- процесс пользователя или среды.

Также система должна работать бесконечно за счёт недетерминированных воздействий среды, возникающих время от времени. Необходимо сформулировать 3 свойства с различной темпоральной структурой и проверить их в nuXmv.

---

## Описание модели

В качестве объекта моделирования был выбран автомат выдачи корма для белок. Данный вариант входит в список систем, предложенных в задании лабораторной работы.

Согласно требованиям лабораторной работы, модель должна описывать параллельную систему и включать процесс управляющего алгоритма, процессы датчиков/индикаторов, процессы актуаторов, а также процесс пользователя или среды. Кроме того, система должна функционировать бесконечно за счёт недетерминированных внешних воздействий.

Построенная модель включает следующие модули:

1. `environment` — модель внешней среды.  
   Этот модуль описывает появление белки у автомата и запрос на пополнение корма. Сигналы среды являются недетерминированными, что позволяет моделировать бесконечную работу системы.

2. `presence_sensor` — датчик присутствия.  
   Он фиксирует факт появления белки и передаёт соответствующий сигнал управляющему модулю.

3. `feeder_controller` — управляющий алгоритм.  
   Контроллер имеет три состояния: ожидание (`idle`), выдача корма (`dispense`) и ожидание ухода белки (`wait_leave`). Если белка обнаружена и корм имеется, контроллер инициирует выдачу корма.

4. `feeder_motor` — исполнительный механизм.  
   Этот модуль моделирует мотор, который непосредственно осуществляет выдачу корма.

5. `low_food_indicator` — индикатор пустого автомата.  
   Когда корм заканчивается, включается сигнальная лампа.

6. `food_storage` — хранилище корма.  
   В модуле задаётся текущее количество корма, а также логические признаки наличия корма, пустого и полного состояния автомата.

Таким образом, модель удовлетворяет структурным требованиям лабораторной работы, поскольку содержит все основные типы компонентов: среду, датчик, контроллер, актуатор и индикатор.

## Структура системы

Система состоит из следующих логических компонентов:
- **процесс среды** — `environment`;
- **процесс датчика** — `presence_sensor`;
- **процесс контроллера** — `feeder_controller`;
- **процесс актуатора** — `feeder_motor`;
- **процесс индикатора** — `low_food_indicator`;
- **процесс хранения ресурса** — `food_storage`.

Таким образом, модель является параллельной и структурно соответствует постановке лабораторной работы.

## Проверяемые свойства

Для верификации модели были сформулированы свойства на языке линейной темпоральной логики LTL. Свойства имеют различную темпоральную структуру, что соответствует требованиям лабораторной работы.

### Свойство 1
```smv
LTLSPEC
  G F env.squirrel_here
```

**белки появляются бесконечно часто.**

Это свойство показывает, что внешняя среда остаётся активной: автомат не переходит в ситуацию, когда белки больше никогда не появляются. Тем самым обеспечивается бесконечное функционирование модели.

### Свойство 2
```smv
LTLSPEC
  G ((sensor.present & stock.has_food & controller.state = idle) ->
     X (controller.state = dispense))
```

**если датчик фиксирует белку, корм имеется, а контроллер находится в режиме ожидания, то на следующем шаге контроллер перейдёт к выдаче корма.**

Это свойство отражает корректность управляющего алгоритма: автомат должен реагировать на обнаружение белки, если в нём есть корм.

### Свойство 3
```smv
LTLSPEC
  G !(motor.motor_on & stock.empty)
```

**мотор никогда не работает, когда корм закончился.**

Это свойство относится к классу свойств безопасности. Оно запрещает некорректную работу исполнительного механизма в ситуации отсутствия ресурса.

### Свойство 4
```smv
LTLSPEC
  G (stock.empty -> F stock.full)
```

**если корм закончился, то в будущем автомат снова станет полным.**

Это свойство относится к свойствам живости. Оно описывает восстановление работы системы после исчерпания корма за счёт моделируемого пополнения запаса.

При запуске `nuXmv` были получены следующие результаты:

```text
-- specification  G ( F env.squirrel_here)  is true
-- specification  G (((sensor.present & stock.has_food) & controller.state = idle) ->  X controller.state = dispense)  is true
-- specification  G !(motor.motor_on & stock.empty)  is true
-- specification  G (stock.empty ->  F stock.full)  is true
```

Все четыре свойства выполняются.
